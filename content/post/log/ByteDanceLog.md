### idGen

1. service scope 唯一而不是表 scope 唯一
2. remote + local
3. remote ：后台 goroutine fetchRemoteId 调 _client.Get(ctx) 持续往容量 200 的 idsChan 灌 ID（ defaultCacheCount = 200 ）。
4. 降级触发 ： select-default 非阻塞读，远端拉不到 / 全部消耗完时， 立即 走 localGet() ，不重试不等待。
5. 高 32 位 ： time.Now().Unix() ，保证不同秒生成的 ID 一定不同。
6. local: 
   - 实例间：低 32 位 中间 19 bit hash 位 ：实例标识， 这是本地降级唯一性的核心 。
   - 实例内：
      1. mutex 保证并发安全
      2. 低 13 bit counter ：每秒最多 8192 个 ID，counter 8192 用满 → 忙等下一秒， 不允许时间戳回退或重复
      3. counter 起点用纳秒随机化，避免 pod 数少时不同秒之间起点完全相同造成可预测
7. remote + local：远端拉到 ID 后会过滤掉 hash 位与本机相同的 ID
